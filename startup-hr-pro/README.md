# StartupHR Pro

> 초기 스타트업 대표를 위한 올인원 HR/법무 관리 플랫폼

![Version](https://img.shields.io/badge/version-1.0.0-blue)
![License](https://img.shields.io/badge/license-MIT-green)

## 🎯 프로젝트 목적

초기 스타트업 대표들이 겪는 HR/법무 문제를 해결하기 위한 웹 애플리케이션입니다.

### 주요 해결 과제
- ❌ 구두 약속만 믿고 계약서를 제대로 작성하지 않음
- ❌ 52시간 초과 근무에 대한 인식 부족
- ❌ 직원과의 감정 소모전 (42분 통화 등)
- ❌ 법적 분쟁 시 증거 자료 부족
- ❌ 교육비 환불, 퇴사 처리 등의 절차 미숙

## ✨ 핵심 기능

### 1. 📊 통합 대시보드
- 한눈에 보는 직원 현황
- 긴급 알림 (52시간 초과, 계약 만료 등)
- 이번 주 근무시간 시각화
- 법적 리스크 점수

### 2. 👥 직원 관리
- 직원 정보 관리
- 직원별 통계 (총 근무시간, 경고 횟수 등)
- 리스크 있는 직원 자동 감지
- 계약 만료 임박 알림

### 3. 📄 계약서 자동 생성
- 법적으로 안전한 계약서 템플릿
- 필수 조항 자동 포함
  - 근로시간 명확화
  - 연장근로 제한
  - 주말 근무 규정
  - 교육비 반환 조항
  - 해지 조건
- 리스크 자동 체크
  - 52시간 초과 계약 차단
  - 최저시급 미달 경고
- PDF/텍스트 다운로드

### 4. ⏰ 근태 관리 (52시간 체크)
- 출퇴근 기록 관리
- 주간 근무시간 자동 계산
- 52시간 초과 실시간 경고
  - 40시간: 🟡 연장근로 중
  - 50시간: 🟠 52시간 임박
  - 52시간: 🔴 위반! 즉시 조치
- 월간 통계 및 엑셀 다운로드

### 5. ⚖️ 법적 리스크 자동 체크
- 5대 리스크 점검
  1. 근로시간 (52시간)
  2. 최저임금
  3. 계약서
  4. 4대보험
  5. 소통 패턴
- 직원별 리스크 점수 (0-100)
- 위험 레벨별 조치사항 제공
- 법적 리포트 다운로드

### 6. 💬 대화 분석
- 대화 내용 리스크 분석
- 감정적 표현 자동 감지
  - "배신", "이기적", "서운" 등
- 법적 리스크 키워드 감지
  - "52시간 초과", "주말도 당연히" 등
- 권장 조치사항 자동 생성
- 10분 룰 적용
- 증거 패키지 생성

### 7. ✅ 일일 체크리스트
- 일일/주간/월간 체크리스트
- 아침/오후/저녁 루틴
- 완료율 자동 계산
- 일일 브리핑 자동 생성
- 상황별 대응 가이드

## 🛠️ 기술 스택

### Frontend
- **Vanilla JavaScript** - 프레임워크 없이 가볍게
- **Tailwind CSS** - 빠른 UI 개발

### Data
- **LocalStorage** - 백엔드 없이 브라우저에 데이터 저장

### Architecture
- **SPA (Single Page Application)** - 페이지 전환 없이 부드러운 UX

## 🚀 시작하기

### 1. 설치 불필요!
이 프로젝트는 **설치가 필요 없습니다**. HTML 파일만 브라우저에서 열면 됩니다.

### 2. 실행 방법

#### 방법 1: 파일 직접 열기
```bash
# 프로젝트 폴더로 이동
cd startup-hr-pro

# index.html을 브라우저에서 열기
# Windows
start index.html

# Mac
open index.html

# Linux
xdg-open index.html
```

#### 방법 2: 로컬 서버 실행 (권장)
```bash
# Python 3가 설치되어 있다면
python -m http.server 8000

# 또는 Node.js가 설치되어 있다면
npx http-server

# 브라우저에서 http://localhost:8000 접속
```

## 📖 사용 가이드

### 첫 실행 시
1. 브라우저에서 `index.html` 열기
2. 자동으로 샘플 데이터가 로드됩니다
   - 직원 3명 (김승용, 이광민, 박현서)
   - 샘플 근태 기록
   - 샘플 케이스 스터디

### 주요 기능 사용법

#### 1. 직원 추가
1. 좌측 메뉴에서 `👥 직원 관리` 클릭
2. `+ 직원 추가` 버튼 클릭
3. 정보 입력 후 저장

#### 2. 계약서 생성
1. `📄 계약서 관리` 메뉴
2. 직원 선택
3. 계약 조건 입력
4. 자동 리스크 체크 확인
5. 계약서 다운로드

#### 3. 근태 기록
1. `⏰ 근태 관리` 메뉴
2. 출퇴근 시간 입력
3. 자동으로 근무시간 계산
4. 52시간 초과 시 즉시 경고

#### 4. 대화 분석
1. `💬 대화 분석` 메뉴
2. 대화 내용 붙여넣기
3. `분석하기` 클릭
4. 리스크 점수 및 키워드 확인
5. 권장 조치사항 확인

#### 5. 법적 리스크 체크
1. `⚖️ 법적 리스크 체크` 메뉴
2. 직원별 리스크 점수 확인
3. 상세 내역 및 조치사항 확인
4. 필요시 리포트 다운로드

### 데이터 관리

#### 백업
- 좌측 하단 `💾 데이터 백업` 클릭
- JSON 파일로 다운로드
- 안전한 곳에 보관

#### 복원
- 백업 파일을 불러오는 기능은 개발자 콘솔에서:
```javascript
// 백업 파일을 읽어서
const backup = JSON.parse('백업파일내용');
db.importAll(backup);
location.reload();
```

#### 초기화
- 좌측 하단 `🗑️ 데이터 초기화` 클릭
- 모든 데이터가 삭제되고 샘플 데이터로 재설정됩니다

## 💡 실전 사용 팁

### 1. 계약서는 항상 서면으로
- ❌ 구두 약속만 믿지 마세요
- ✅ 계약서 생성 기능을 사용해 서면화하세요

### 2. 52시간은 절대 선
- 🟡 40시간 넘으면 주의
- 🟠 50시간 넘으면 경고
- 🔴 52시간 넘으면 즉시 조치

### 3. 대화는 10분 이내로
- 10분 넘어가면 → 즉시 종료
- 내용을 서면으로 정리해서 발송

### 4. 감정적 표현 절대 금지
- ❌ "배신감", "서운하다", "이기적"
- ✅ "기준 미달", "개선 필요", "규정 위반"

### 5. 모든 것을 기록하세요
- 면담 내용
- 경고 사항
- 칭찬 내용
- 소통 기록
- 나중에 법적 증거가 됩니다

## 🎓 실전 케이스 스터디

### Case 1: 초기 열정 → 급격한 의욕 저하

**상황**
- 입사 초기: "다 하겠다", "24시간도 가능"
- 2개월 후: 주말 거부, 소통 회피, 태도 급변

**문제점**
- 구두 약속만 믿음
- 42분 감정 소모전
- 같은 설명 10번 반복

**올바른 대응**
1. ✅ 초기에 과도한 약속 받지 않기
2. ✅ 모든 조건 변경 서면화
3. ✅ 소통 회피 3회 시 경고
4. ✅ 대화 10분 넘으면 종료
5. ✅ 조기 정리 결단

### Case 2: 교육비 지원 후 단기 퇴사

**예방 방법**
- 계약서에 환불 조항 명시
- 비율, 기간, 조건 구체적으로
- "자발적 동의" 명시

## 🔐 보안 및 프라이버시

### 데이터 저장 위치
- 모든 데이터는 **브라우저의 LocalStorage**에만 저장됩니다
- 외부 서버로 전송되지 않습니다
- 브라우저를 닫아도 데이터는 유지됩니다

### 주의사항
- LocalStorage는 브라우저별로 독립적입니다
- 브라우저 캐시를 지우면 데이터가 삭제됩니다
- 정기적으로 백업하세요

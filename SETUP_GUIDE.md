# 브링엔지니어링 경영관리 시스템 - 설치 가이드

## 1. 사전 준비

### Node.js 설치
1. https://nodejs.org 접속
2. **LTS 버전** 다운로드 및 설치 (Next → Next 기본 설정으로 완료)
3. 설치 확인: 명령 프롬프트(cmd)에서 아래 명령어 입력
```
node -v
```
`v20.x.x` 같은 버전이 출력되면 성공


## 2. 시스템 설치

### 방법 A: ZIP 다운로드 (간편)
1. GitHub 레포지토리에서 **Code → Download ZIP** 클릭
2. 다운받은 ZIP 파일을 원하는 위치에 압축 해제
3. 압축 해제된 폴더 이름은 자유롭게 변경 가능 (예: `브링엔지니어링`)

### 방법 B: Git Clone (권장)
```
git clone <레포지토리 URL> 브링엔지니어링
```


## 3. 서버 실행

### Windows (간편 실행)
1. 폴더 안의 `start.bat` 파일을 더블클릭
2. 자동으로 의존성 설치 + 서버 시작 + 브라우저 열림

### Windows (수동 실행)
```
cd 폴더경로
npm install
node server.js
```

### 브라우저 접속
- 서버 실행 후 브라우저에서: http://localhost:3000


## 4. 같은 네트워크에서 COO와 함께 사용하기

### 서버 담당자 (대표님 PC)

#### Step 1: 내 PC의 IP 주소 확인
명령 프롬프트(cmd)에서:
```
ipconfig
```
→ `IPv4 주소` 항목 확인 (예: `192.168.0.10`)

#### Step 2: 서버 실행
```
node server.js
```

#### Step 3: Windows 방화벽에서 포트 열기
1. Windows 검색 → "Windows Defender 방화벽" → "고급 설정"
2. 좌측 "인바운드 규칙" → 우측 "새 규칙"
3. 설정:
   - 규칙 종류: **포트**
   - TCP, 특정 로컬 포트: **3000**
   - 연결 허용
   - 이름: `브링엔지니어링 서버`
4. 확인

### 접속자 (COO)
1. 같은 Wi-Fi / 사내 네트워크에 연결
2. 브라우저에서 접속:
```
http://192.168.0.10:3000
```
(IP 주소는 서버 담당자가 알려준 주소로 변경)


## 5. 비밀번호 설정 (보안)

1. 시스템 접속 후 설정 메뉴에서 비밀번호 설정
2. 비밀번호 설정 시 모든 접속자가 로그인 필요
3. 서버 담당자와 COO가 같은 비밀번호로 로그인


## 6. 데이터 관리

| 항목 | 위치 |
|------|------|
| 앱 데이터 | `data/app-data.json` |
| 업로드 문서 | `data/documents/` |
| 백업 파일 | `data/backup-*.json` |

- 시스템 내 **백업** 기능으로 수시 백업 권장
- `data/` 폴더를 통째로 복사해두면 전체 백업 가능

```

│   ├── 03_인사조직/
```

## 8. 문제 해결

| 증상 | 해결 방법 |
|------|-----------|
| `node`를 찾을 수 없음 | Node.js 재설치 후 PC 재시작 |
| `npm install` 오류 | `node_modules` 폴더 삭제 후 다시 `npm install` |
| 다른 PC에서 접속 안됨 | 방화벽 포트 3000 개방 확인 |
| 포트 3000 이미 사용 중 | `set PORT=3001` 후 `node server.js` (포트 변경) |
| 데이터가 안 보임 | `data/` 폴더 존재 확인 |

# 브링엔지니어링 — 품질관리체계 (QMS)

**최종 수정일:** 2026-03-10
**참고:** ISO 9001:2015 (스타트업 경량 적용), KOLAS 인정 기준

---

## 품질 관리 원칙

1. **고객 만족**: 납품물이 고객 기대를 충족하는가가 최종 기준
2. **프로세스 기반**: 결과뿐 아니라 과정도 관리
3. **지속 개선**: 매 납품 후 회고 → 프로세스 개선
4. **증거 기반**: "잘했다"가 아니라 "테스트 결과가 기준을 충족했다"

## Shadow AI 모델 품질 기준

| 항목 | 기준 | 측정 방법 |
|------|------|----------|
| 조도 측정 정확도 | ≥95% | 공인 측정 장비와 비교 |
| 실시간 처리 속도 | ≤1초/프레임 | 벤치마크 테스트 |
| False Positive Rate | ≤3% | 테스트 데이터셋 |
| False Negative Rate | ≤2% | 테스트 데이터셋 |
| 모델 안정성 | 24시간 연속 운영 무중단 | 스트레스 테스트 |

## 납품 프로세스

```
개발 완료 → 내부 테스트 → QA 체크리스트 → 수정 → 최종 검증 → 납품 → 고객 확인
```

### 단계별 책임

| 단계 | 담당 | 산출물 |
|------|------|--------|
| 내부 테스트 | 개발자 | 테스트 결과서 |
| QA 체크리스트 | 리뷰어 (교차 검증) | 체크리스트 완료본 |
| 최종 검증 | 대표 | 납품 승인 |
| 납품 | 대표 | 납품 확인서 |
| 고객 확인 | 고객 | 검수 확인서 |

## 고객 불만/이슈 대응

| 등급 | 기준 | 대응 시간 | 해결 기한 |
|------|------|----------|----------|
| 긴급 | 서비스 중단, 데이터 오류 | 2시간 | 24시간 |
| 높음 | 주요 기능 오작동 | 4시간 | 3일 |
| 보통 | 사소한 버그, UI 이슈 | 1일 | 1주 |
| 낮음 | 개선 요청, 문의 | 2일 | 다음 업데이트 |

## 품질 개선 사이클 (PDCA)

```
Plan (계획) → Do (실행) → Check (점검) → Act (개선)
```

매 납품 후:
1. **회고 미팅** (30분): 잘한 것, 아쉬운 것, 개선할 것
2. **QMS 업데이트**: 새로 발견된 체크 항목 추가
3. **프로세스 개선**: 반복 문제에 대한 구조적 해결

# 브링엔지니어링 — 성과평가 체계

**최종 수정일:** 2026-03-10
**참고:** Google OKR, Netflix Keeper Test, Radical Candor

---

## 평가 원칙

1. **공정성**: 사전에 합의한 기준으로만 평가
2. **성장 지향**: 평가는 서열이 아닌 성장 도구
3. **2축 평가**: 성과(What) + 핵심가치(How) 동시 평가
4. **투명성**: 평가 기준과 결과를 본인에게 공개

## 2축 평가 프레임워크

### 축 1: 성과 (What) — 70%

| 등급 | 기준 | 설명 |
|------|------|------|
| S (탁월) | OKR 0.8+ & 기대 이상의 성과 | 팀 전체에 임팩트 |
| A (우수) | OKR 0.6~0.8 & 안정적 기여 | 맡은 일을 잘 해냄 |
| B (보통) | OKR 0.4~0.6 | 기본은 하나 개선 필요 |
| C (미흡) | OKR 0.4 미만 | 심각한 개선 필요 |

### 축 2: 핵심가치 실천 (How) — 30%

| 핵심가치 | 평가 질문 |
|---------|----------|
| 임팩트 우선 | 중요한 일에 집중했는가? 우선순위를 잘 판단했는가? |
| 투명한 공유 | 진행 상황, 이슈를 적시에 공유했는가? |
| 주인의식 | 맡은 일에 끝까지 책임졌는가? 범위 밖도 도왔는가? |
| 빠른 실험 | 완벽주의에 빠지지 않고 빠르게 실행했는가? |
| 서로 존중 | 팀원에게 건설적 피드백을 주고받았는가? |

### 2축 매트릭스

```
핵심가치 실천
    높음 │ "문화 리더"     │  "스타 플레이어"  │
         │ (성과 보통,     │  (성과 좋고      │
         │  가치 우수)     │   가치도 우수)    │
    ─────┼────────────────┼─────────────────┤
    낮음 │ "개선 필요"     │  "성과만 좋음"   │
         │ (둘 다 낮음)    │  (독단적 위험)   │
         └────────────────┴─────────────────┘
              낮음                높음
                    성과 달성
```

> **Netflix 원칙 적용:** "뛰어난 성과를 내지만 가치를 해치는 사람"은 팀에 독.
> 성과와 가치 모두 충족해야 진정한 우수 평가.

## 반기 평가 프로세스

### Step 1: 자기 평가 (평가 1주 전)
팀원이 직접 작성:
- 이번 반기 주요 성과 3가지
- OKR 달성 점수 (자체 평가)
- 핵심가치 실천 자기 점검
- 다음 반기 성장 목표

### Step 2: 동료 피드백 (선택, 팀 5명 이상 시 도입)
- "이 사람과 함께 일하면서 가장 좋았던 점"
- "이 사람이 더 잘할 수 있는 점"

### Step 3: 대표 평가
- 자기 평가 + (동료 피드백) 참고하여 2축 평가
- 구체적 근거와 함께 등급 부여

### Step 4: 피드백 면담 (1시간)
1. 본인 회고 공유 (15분)
2. 대표 평가 결과 공유 (15분)
3. 건설적 대화 (15분)
4. 다음 반기 목표 합의 (15분)

## Keeper Test (Netflix 방식, 참고용)

> "이 사람이 다른 회사로 떠난다고 하면, 전력을 다해 만류할 것인가?"

- **Yes** → 우수 인재. 더 좋은 기회와 보상을 제공
- **No** → 솔직한 피드백 + 개선 기회 부여 (3개월)
- 3개월 후에도 개선 없으면 → 정중하게 이별

> ⚠️ 초기 스타트업에서는 이 테스트를 **너무 엄격하게 적용하지 않을 것**.
> 대학생 팀 특성상 성장 잠재력을 더 많이 봐야 함.

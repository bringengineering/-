# 브링엔지니어링 — 보상·인센티브 체계

**최종 수정일:** 2026-03-10
**참고:** Stripe 초기 보상 철학, Buffer 급여 투명 모델

---

## 현실 인식

> 브링엔지니어링은 **매출 미발생 상태**의 초기 스타트업입니다.
> 따라서 보상 체계는 **성장 단계에 따라 진화**합니다.
> 지금 없는 돈을 약속하지 않되, 미래의 보상을 공정하게 설계합니다.

---

## 성장 단계별 보상 설계

### Stage 1: 현재 (매출 0원) — "인정과 성장"

금전적 보상이 어려운 단계. **비금전적 보상**에 집중.

| 보상 유형 | 내용 |
|----------|------|
| **역할 인정** | 공식 직함 부여 (예: "AI Tech Lead") |
| **스킬 레벨 인정** | 모듈별 초급→고급→마스터 등급 공식 인정 |
| **학습 기회** | 주 4시간 자기개발 시간 보장 |
| **외부 발표 기회** | 컨퍼런스, 세미나 발표 지원 |
| **의사결정 참여** | 핵심 의사결정에 팀원 의견 반영 |
| **유연 근무** | 코어타임 외 자율 근무 |

### Stage 2: 첫 매출 발생 — "성과급 시작"

| 보상 유형 | 내용 |
|----------|------|
| **프로젝트 성과급** | 프로젝트 수익의 일정 비율을 팀에 배분 |
| **분기 보너스** | OKR 달성률 70% 이상 시 지급 |
| **장비 지원** | 업무용 장비 업그레이드 |

### Stage 3: 안정적 매출 (연 매출 3억+) — "체계적 보상"

| 보상 유형 | 내용 |
|----------|------|
| **정기 급여** | 시장 수준의 80~100% |
| **반기 성과급** | 평가 등급별 차등 (S: 15%, A: 10%, B: 5%) |
| **스톡옵션** | 핵심 인재 대상 부여 |

### Stage 4: 성장기 (연 매출 10억+) — "시장 경쟁력"

| 보상 유형 | 내용 |
|----------|------|
| **시장 수준 급여** | 동종업계 상위 25% |
| **스톡옵션 확대** | 전 직원 대상 |
| **복리후생** | 건강검진, 교육비, 식대 등 |

## 스킬 레벨 시스템

### 등급 체계

| 레벨 | 명칭 | 기준 | 인정 방법 |
|------|------|------|----------|
| Lv.1 | 학습자 | 기본 개념 이해, 가이드 필요 | 자동 (입사 시) |
| Lv.2 | 실행자 | 독립적 업무 수행 가능 | 30일 온보딩 완료 |
| Lv.3 | 숙련자 | 복잡한 문제 해결, 품질 높은 산출물 | 분기 리뷰에서 인정 |
| Lv.4 | 전문가 | 해당 영역 최고 수준, 타인 멘토링 | 반기 평가에서 인정 |
| Lv.5 | 마스터 | 외부에서도 인정받는 전문성 | 외부 발표/특허/논문 |

### 레벨업 트리거

- Lv.3 달성 → **모듈 마스터** 자격 부여
- Lv.4 달성 → **팀 리드** 역할 확장 기회
- Lv.5 달성 → **CTO/VP급** 직함 + 의사결정 권한 확대

## 스톡옵션 가이드라인 (Stage 3 이후)

| 항목 | 기준 |
|------|------|
| 대상 | 정규 멤버, 6개월 이상 근속 |
| 규모 | 초기 멤버: 전체의 1~3%, 이후 합류: 0.1~1% |
| 베스팅 | 4년 베스팅, 1년 클리프 |
| 행사 조건 | 투자 유치 또는 매출 일정 수준 달성 시 |

> ⚠️ 스톡옵션은 **법적 자문 후** 정식 계약으로 진행.
> 구두 약속은 효력 없음.

## 비금전적 보상 — 지금 당장 할 수 있는 것

### 1. 공개 인정 (Weekly Shoutout)
주간 리뷰에서 이번 주 최고의 기여자를 공개 칭찬

### 2. 역할 타이틀 부여
- 명함, 이메일 서명, LinkedIn에 사용 가능한 공식 직함
- 예: "AI Engineer", "Lead SW Developer"

### 3. 학습 투자
- 주 4시간 자기개발 시간 (Google의 20% 타임 축소 버전)
- 온라인 강의, 도서 구매 지원 (월 5만원 한도)

### 4. 의미 부여
- 본인의 기여가 제품/회사에 미친 임팩트를 구체적으로 전달
- "네가 만든 이 기능 덕분에 고객 데모가 성공했다"

## 급여 산정 공식 (Stage 3 이후)

```
월 급여 = 기본급 × 역할 계수 × 경력 계수

기본급: 회사가 정한 기준액
역할 계수: 주니어(1.0), 시니어(1.3), 리드(1.5), VP(1.8)
경력 계수: 매년 +5% (최대 +30%)
```

> Buffer의 투명 급여 공식 참고.
> 공식을 공개하여 급여 불만/불공정 해소.

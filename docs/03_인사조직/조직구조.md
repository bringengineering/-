## R&R (역할과 책임) 매트릭스

| 역할 | 서창환 (대표) | 노광민 (SW) | 배정환 (AI) | 김현서 |
|------|:---:|:---:|:---:|:---:|
| 경영 전략/의사결정 | **R** | I | I | I |
| 사업 개발/영업 | **R** | S | S | S |
| 정부과제 관리 | **R** | S | S | S |
| VisiScan 개발 | A | **R** | S | - |
| Shadow AI 개발 | A | S | **R** | - |
| 백엔드/서버 | I | **R** | S | - |
| AI 모델/데이터 | I | S | **R** | - |
| 재무/회계 | **R** | - | - | S |
| 대외 커뮤니케이션 | **R** | S | S | S |
| 문서/체계 관리 | A | S | S | S |

> **R** = Responsible (실행), **A** = Accountable (최종책임), **S** = Support (지원), **I** = Informed (통보), **-** = 해당없음

## 모듈 마스터 제도

각 영역에 **한 명의 마스터**를 지정하여 책임감과 전문성 강화

| 모듈 | 마스터 | 역할 |
|------|--------|------|
| 백엔드 시스템 | 노광민 | API, DB, 서버 인프라 |
| AI 파이프라인 | 배정환 | 모델 학습, 추론, MLOps |
| 비즈니스/영업 | 서창환 | 고객, 계약, 전략 |
| 인프라/DevOps | 노광민 | CI/CD, 클라우드, 배포 |
| 데이터 관리 | 배정환 | 데이터 수집, 전처리, 품질 |

## 조직 성장 로드맵 (Spotify Squad Model 참고)

### Phase 1: 현재 (4명) — "원팀"
```
하나의 팀 = 하나의 Squad
전원이 모든 프로젝트에 참여
대표가 Product Owner 겸임
```

### Phase 2: 7~10명 — "듀얼 스쿼드"
```
Squad 1: VisiScan + Shadow AI (제품팀)
Squad 2: Lux-Guard + FDI (연구팀)
각 Squad에 리드 1명 + 멤버 3~4명
```

### Phase 3: 15~20명 — "트라이브"
```
Tribe: 인프라 안전 솔루션
├── Squad 1: VisiScan (서비스)
├── Squad 2: Shadow AI (AI엔진)
├── Squad 3: 하드웨어 (Lux-Guard/FDI)
└── Chapter: AI, Backend, Frontend (기술 교류 그룹)
```

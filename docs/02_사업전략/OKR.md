# 브링엔지니어링 — OKR 목표관리 체계

**최종 수정일:** 2026-03-10
**현재 분기:** 2026 Q2 (4~6월)
**참고:** Google OKR, "Measure What Matters" (존 도어)

---

## OKR 운영 규칙

### 기본 원칙
1. **회사 OKR**: 분기당 Objective 3~4개, 각 KR 2~3개
2. **개인 OKR**: 회사 OKR에서 파생, Objective 2~3개
3. **점수**: 0.0~1.0 (Google 방식, 0.6~0.7이 건강한 목표)
4. **주기**: 분기 설정 → 주간 체크인 → 분기말 점수 매기기

### 주간 체크인 (매주 금요일 주간리뷰에서)
- 각 KR의 현재 진행률 업데이트 (%)
- 블로커/리스크 공유
- 필요시 KR 수정 (O는 분기 내 변경 불가)

## OKR 점수 기준 (Google 방식)

| 점수 | 의미 | 해석 |
|------|------|------|
| 1.0 | 완벽 달성 | 목표가 너무 쉬웠을 수 있음 |
| 0.7~0.9 | 우수 | 도전적이고 좋은 성과 |
| **0.6~0.7** | **건강한 목표** | **이 범위가 이상적** |
| 0.4~0.6 | 부분 달성 | 블로커 분석 필요 |
| 0.0~0.3 | 미달 | 목표 설정 또는 역량 문제 점검 |

> **중요:** OKR 점수는 성과평가/보상과 **직접 연동하지 않음** (Google 원칙).
> OKR은 도전적 목표 설정 도구이지, 성과 측정 도구가 아님.
> 성과평가는 별도 기준 사용 → `03_인사조직/성과평가.md`

## OKR 분기 사이클

```
분기 마지막 2주: 이번 분기 OKR 점수 매기기 + 회고
        ↓
분기 마지막 1주: 다음 분기 회사 OKR 초안 (대표)
        ↓
분기 첫 1주: 전체 워크숍에서 확정 + 개인 OKR 설정
        ↓
매주: 주간 리뷰에서 KR 진행률 체크인
        ↓
매월: 월간 경영 리뷰에서 OKR 전체 점검
```

## 파이프라인 단계 정의

```
[1단계] 리드 발굴 → [2단계] 접촉/미팅 → [3단계] 니즈 파악 → [4단계] 제안서 → [5단계] 협상 → [6단계] 계약 → [7단계] 납품/서비스
```

| 단계 | 정의 | 전환율 기준 | 담당 |
|------|------|-----------|------|
| 1. 리드 발굴 | 잠재 고객 식별 | - | 대표/전원 |
| 2. 접촉/미팅 | 첫 미팅 또는 연락 | 50% | 대표 |
| 3. 니즈 파악 | 고객 요구사항 구체화 | 60% | 대표 + 기술팀 |
| 4. 제안서 | 제안서/견적서 발송 | 70% | 대표 |
| 5. 협상 | 조건 조율 | 80% | 대표 |
| 6. 계약 | 계약서 체결 | 95% | 대표 |
| 7. 납품 | 서비스 수행/납품 | 100% | 기술팀 |

## 제안서 작성 가이드

### 구조 (A4 10장 이내)

1. **표지** — 프로젝트명, 고객사명, 제안일
2. **요약** (1장) — 핵심 제안 한눈에
3. **고객 현황/문제** (1장) — 고객이 겪는 문제 정의
4. **솔루션** (2~3장) — 브링의 해결 방안
5. **실행 계획** (1장) — 일정, 단계, 산출물
6. **팀 소개** (1장) — 수행 역량, 레퍼런스
7. **견적** (1장) — 비용 구조
8. **부록** — 기술 상세, 특허, 수상 내역

### 핵심 원칙
- 고객 입장에서 작성 (기술 자랑 X, 고객 이익 O)
- 숫자로 말하기 (비용 절감 50%, 소요 시간 3일 등)
- 경쟁사 비교 포함 (객관적으로)

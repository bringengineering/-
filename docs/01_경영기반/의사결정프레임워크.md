# 브링엔지니어링 — 의사결정 프레임워크

**최종 수정일:** 2026-03-10
**참고:** RAPID (Bain), RACI, Amazon Leadership Principles

---

## 왜 필요한가?

대표 1인에게 모든 결정이 집중되면:
- 대표가 병목이 됨 → 속도 저하
- 팀원이 수동적으로 변함 → 주인의식 하락
- 대표 부재 시 조직이 멈춤

**목표:** 명확한 위임 기준을 세워, 각자가 자기 영역에서 빠르게 결정하되, 중요한 것만 대표가 판단

---

## 의사결정 4단계

### Level 1: 자율 결정 (Just Do It)

| 항목 | 내용 |
|------|------|
| **기준** | 본인 업무 범위 내, 비용 0원, 타인 영향 없음 |
| **결정자** | 담당자 본인 |
| **보고** | 불필요 (데일리 스탠드업에서 자연 공유) |

**예시:**
- 코드 구조 변경, 리팩토링
- 문서 작성/수정
- 개인 학습 계획 조정
- 디자인 시안 수정

### Level 2: 공유 후 실행 (Inform & Execute)

| 항목 | 내용 |
|------|------|
| **기준** | 본인 업무 범위이나, 다른 팀원에게 영향 있음 |
| **결정자** | 담당자 (관련자에게 사전 공유) |
| **보고** | 슬랙/카톡으로 관련자에게 공유 후 24시간 이의 없으면 실행 |

**예시:**
- API 스펙 변경 (프론트↔백엔드 영향)
- 일정 1~2일 조정
- 외부 미팅 일정 변경
- 기술 스택 라이브러리 추가

### Level 3: 합의 후 실행 (Discuss & Decide)

| 항목 | 내용 |
|------|------|
| **기준** | 2명 이상 영향, 비용 50만 원 미만 |
| **결정자** | 관련자 합의 → 대표 보고 |
| **보고** | 주간 킥오프 또는 슬랙에서 논의 후 결정 기록 |

**예시:**
- 외주 발주 (50만 원 미만)
- 장비/소프트웨어 구매
- 프로젝트 마일스톤 변경
- 신규 협업 도구 도입

### Level 4: 대표 승인 (CEO Approval)

| 항목 | 내용 |
|------|------|
| **기준** | 비용 50만 원 이상, 대외 계약, 인사 결정 |
| **결정자** | 대표 최종 승인 |
| **보고** | 의사결정 요청서 작성 → 대표 결재 |

**예시:**
- 채용/퇴직
- 고객 계약 체결
- 투자 유치/IR
- 정부과제 신규 지원
- 50만 원 이상 지출
- 회사 전략 방향 변경
- 특허 출원/법률 관련

## 의사결정 요청서 (Level 3~4용)

Level 3~4 결정 시 아래 양식으로 공유:

```
## 의사결정 요청

**제목:** [결정할 사항]
**요청자:** [이름]
**긴급도:** [상/중/하]
**기한:** [결정 필요 시점]

### 배경
[왜 이 결정이 필요한가?]

### 선택지
| 선택지 | 장점 | 단점 | 비용 |
|--------|------|------|------|
| A안 | | | |
| B안 | | | |

### 추천안
[추천하는 선택지와 이유]
```

> **Amazon 스타일 원칙:** 결정을 두 종류로 나눈다.
> - **One-way door** (되돌리기 어려움): 신중하게 → Level 4
> - **Two-way door** (되돌리기 쉬움): 빠르게 → Level 1~2

---

## 의사결정 속도 기준

| 긴급도 | 응답 기한 | 채널 |
|--------|----------|------|
| 🔴 긴급 | 2시간 이내 | 전화/카카오톡 |
| 🟡 중요 | 24시간 이내 | 슬랙/카톡 |
| 🟢 일반 | 주간 킥오프에서 | 문서/슬랙 |

---

## 교착 상태 해결 규칙

합의가 안 될 때:

1. **30분 토론** → 합의 안 되면
2. **각자 의견을 1장으로 정리** → 24시간 내 공유
3. **대표가 최종 결정** (Amazon의 "Disagree and Commit")
4. 결정 후에는 **전원이 그 결정을 100% 지지하고 실행**

---

## 결정 기록 원칙

> **"기록되지 않은 결정은 결정이 아니다"**

모든 Level 3~4 결정은 다음을 기록:
- 무엇을 결정했는가
- 왜 그렇게 결정했는가
- 누가 참여했는가
- 언제까지 실행하는가
